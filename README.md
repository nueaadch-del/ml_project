# 뇌졸중 환자 발생 예측 모델 만들기

## 1. 프로젝트 소개

본 프로젝트는 건강 데이터를 바탕으로 뇌졸중 발생 가능성을 예측하는 머신러닝 모델을 만드는 것을 목표로 한다.
나이, 고혈압 여부, 평균 혈당 수치, BMI, 흡연 상태, 직업 등 다양한 건강 정보를 활용하여 뇌졸중 발생 여부를 예측하였다.

뇌졸중 예측 문제는 단순히 정확도만 높은 모델을 만드는 것보다, 실제 뇌졸중 환자를 얼마나 잘 찾아내는지가 중요하다. 따라서 본 프로젝트에서는 Accuracy뿐만 아니라 Recall, F1-score, ROC-AUC 등 여러 평가 지표를 함께 확인하였다.

---

## 2. 데이터셋 설명

사용한 데이터셋은 뇌졸중 발생 여부와 관련된 건강 정보가 포함된 데이터이다.

### 주요 변수

| 변수명               | 설명        |
| ----------------- | --------- |
| age               | 나이        |
| hypertension      | 고혈압 여부    |
| avg_glucose_level | 평균 혈당 수치  |
| bmi               | 체질량지수     |
| smoking_status    | 흡연 상태     |
| work_type         | 직업 유형     |
| stroke            | 뇌졸중 발생 여부 |

### 결측치 처리

* `bmi` 결측치: BMI 평균값으로 대체하였다.
* `age` 결측치: 테스트 데이터에서 결측치가 있는 행을 제거하였다.

---

## 3. 사용한 모델

여러 머신러닝 모델을 학습하고 성능을 비교하였다.

* Logistic Regression
* KNN
* Decision Tree
* Random Forest
* AdaBoost
* XGBoost

여러 모델의 성능을 비교한 결과, 최종적으로 **XGBoost 모델**을 선택하였다.

---

## 4. 모델 학습 및 성능 개선 과정

데이터셋은 뇌졸중 환자보다 뇌졸중이 아닌 사람의 비율이 훨씬 높은 불균형 데이터였다.
이러한 경우 모델이 모든 사람을 “뇌졸중이 아니다”라고 예측해도 Accuracy가 높게 나올 수 있다.

하지만 뇌졸중 예측에서는 실제 환자를 놓치는 것이 더 위험하다고 판단하였다.
따라서 Accuracy만 보지 않고, Recall과 F1-score도 함께 확인하였다.

성능 개선을 위해 다음과 같은 과정을 진행하였다.

* 여러 머신러닝 모델 학습 및 비교
* 하이퍼파라미터 조정
* 데이터 불균형을 줄이기 위한 `balanced` 적용
* Threshold 조정 실험
* ROC-AUC Curve를 통한 분류 성능 확인

---

## 5. 주요 결과

Feature Importance를 확인한 결과, 뇌졸중 예측에 가장 큰 영향을 준 변수는 **age**였다.
즉, 나이가 모델의 예측 과정에서 가장 중요한 변수로 사용되었다.

Threshold 조정을 진행했지만, 큰 성능 향상은 찾기 어려웠다.
최종적으로는 ROC-AUC Curve를 통해 모델별 분류 성능을 비교하였고, 여러 평가 결과를 종합하여 **XGBoost 모델**을 최종 모델로 선택하였다.

---

## 6. 결론

본 프로젝트를 통해 뇌졸중 예측에서는 단순히 Accuracy가 높은 모델이 좋은 모델이라고 보기 어렵다는 것을 알 수 있었다.

데이터셋이 불균형한 경우, 모델이 대부분의 데이터를 다수 클래스인 “뇌졸중 아님”으로 예측해도 정확도가 높게 나타날 수 있다.
그러나 이는 실제 뇌졸중 환자를 제대로 찾아냈다는 의미가 아니기 때문에 Recall과 F1-score를 함께 확인하는 것이 중요하다.

여러 모델을 비교한 결과, XGBoost 모델이 가장 괜찮은 성능을 보여 최종 모델로 선택하였다.

---

## 7. 개선 방향

현재 데이터셋은 전체 데이터 양이 많지 않고, 뇌졸중 환자 데이터의 비율도 낮다는 한계가 있었다.
앞으로는 다른 뇌졸중 관련 데이터셋을 추가로 가져와 기존 데이터와 병합한다면 더 좋은 결과를 얻을 수 있을 것이다.

또한 혈압, 가족력, 운동량, 식습관 등 더 다양한 건강 정보를 추가하면 모델이 더 많은 특징을 학습할 수 있다.
추가적으로 딥러닝 모델을 사용하여 기존 머신러닝 모델과 성능을 비교해 보는 것도 좋은 개선 방향이 될 수 있다.
